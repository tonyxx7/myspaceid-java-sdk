Java SDK for MySpaceID

This SDK may be used in the following situations:
  1. External Website: You have a website and you want your users to be able to log on using their MySpace accounts. After a user logs on, this SDK will allow you to fetch his/her data such as name, friends, etc., from MySpace. See below for options on implementing authentication.
  2. On-site App: You have a MySpace on-site application and you need to write server-side code that accesses user data. This SDK allows you to access user data without the need for authentication.

In the first case, you will need to authenticate the user. In the second case, authentication is not necessary, and you will be able to fetch information about any user who has installed your application.

Authentication for Your External Website

For case 1, you have two choices for how to implement authentication:

  • OAuth: use the classes documented in this Javadoc. Refer to sample/oauth to see how this API is used. To run this sample, please enter your own key/secret in oauth-demo.jsp.
  • OpenID-OAuth hybrid: customize the sample in the sample/hybrid directory. To customize, replace hybrid-login.jsp with your own page to invoke the consumer servlet, and replace return.jsp to pull user data after the user has logged on. Note that return.jsp is specified by the "return" servlet initialization parameter in web.xml. To get this sample to run, you will need to:
    • Enter your own key/secret in web.xml as well as return.jsp
    • Enter your own key in web.xml
